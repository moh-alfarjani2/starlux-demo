بنية مشروع Starlux - (Frontend Demo)
=====================================

هذا المشروع هو تطبيق "فرونت إند" فقط (Frontend-only) يعتمد على React و Vite، وليس Next.js. تم تحويله بالكامل ليعمل كديمو تفاعلي يعتمد على Client-side Routing.

الهيكل العام للمجلدات:
-------------------

/ (الجذر)
├── index.html          # نقطة الدخول الأساسية للمتصفح.
├── package.json        # ملف التبعيات والإعدادات (dependencies).
├── tsconfig.json       # إعدادات TypeScript.
├── vite.config.ts      # إعدادات أداة البناء Vite.
├── tailwind.config.ts  # إعدادات التصميم (Tailwind CSS).
├── postcss.config.mjs  # إعدادات PostCSS لـ Tailwind.
└── public/             # الملفات الثابتة (الصور، الفافيكون).

/src (مجلد الكود المصدري)
├── main.tsx            # تشغيل التطبيق (Entry Point) وربطه بـ index.html.
├── App.tsx             # إعدادات التوجيه الأساسية (React Router).
│
├── /app                # مكونات الصفحات (تستخدم داخل App.tsx)
│   ├── page.tsx        # الصفحة الرئيسية.
│   ├── /hotels         # صفحات الفنادق واستعراضها.
│   │   └── /view       # عرض تفاصيل الفندق (النسخة المتوافقة مع Vite).
│   ├── /dashboard      # لوحة تحكم المستخدم.
│   ├── /booking        # عملية الحجز.
│   ├── /admin          # واجهات الإدارة (محاكاة).
│   └── globals.css     # الأنماط العامة وتكوينات Tailwind.
│
├── /components         # المكونات القابلة لإعادة الاستخدام
│   ├── /ui             # مكونات واجهة المستخدم الأساسية (أزرار، مدخلات، بطاقات).
│   ├── /layout         # مكونات الهيكل العام (Navbar, BaseLayout).
│   ├── /hotel          # مكونات خاصة بعرض تفاصيل الفنادق.
│   └── /sections       # أقسام كبيرة في الصفحات (Hero section, Featured).
│
├── /context            # إدارة حالة التطبيق (State Management)
│   └── starlux-context.tsx # مخزن البيانات العام (المفضلة، المستخدم، الحجوزات).
│
├── /lib                # الأدوات والبيانات الوهمية
│   ├── mock-data.ts    # قاعدة البيانات الوهمية (الفنادق، الصور، التقييمات).
│   ├── db.ts           # محاكاة عمليات قاعدة البيانات وحفظ البيانات في LocalStorage.
│   └── utils.ts        # دوال مساعدة عامة.

آلية العمل الحالية:
---------
- التوجيه (Routing): يتم بالكامل عبر `react-router-dom` في `App.tsx`.
- البيانات: يتم تخزين أي بيانات (مثل الحجوزات أو المفضلة) في متصفح المستخدم عبر LocalStorage.
- الحالة: يتم توزيع البيانات عبر React Context لضمان مزامنة الواجهة في كل الصفحات.
- البناء: تم حذف كل ملفات Next.js وإصلاح استيرادات المكتبات لتتوافق مع بيئة Vite الصافية.
