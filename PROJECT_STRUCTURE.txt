دليل هيكلية مشروع Starlux - النسخة التفصيلية الكاملة (Frontend Demo)
==========================================================

هذا المستند يقدم شرحاً شاملاً لكل جزء من أجزاء مشروع Starlux، مصمماً ليكون مرجعاً تقنياً دقيقاً يوضح كيفية بناء وتشغيل المشروع كنسخة "ديمو" احترافية تعتمد كلياً على الواجهة الأمامية (Frontend).

1. المجلدات والملفات في الجذر (Root Directory)
-------------------------------------------
* index.html: 
    - الملف الأساسي ونقطة الانطلاق للمتصفح.
    - يحتوي على وسم <div id="root"> حيث يتم حقن تطبيق React بالكامل.
* package.json:
    - عصب المشروع؛ يحتوي على قائمة المكتبات (Dependencies) مثل React و Vite و Tailwind.
    - يحدد "السكربتات" مثل `npm run dev` للتشغيل و `npm run build` للبناء.
* vite.config.ts:
    - ملف إعدادات Vite، الأداة المسؤولة عن بناء وتشغيل المشروع بسرعة فائقة.
* tsconfig.json & tsconfig.node.json:
    - إعدادات لغة TypeScript لضمان كتابة كود برمجى خالٍ من الأخطاء النوعية (Types).
* tailwind.config.ts & postcss.config.mjs:
    - ملفات إعدادات إطار العمل Tailwind CSS المسؤولة عن كل جماليات وتصميم الموقع.
* README.md:
    - ملف تعريفي عام بالمشروع للمطورين.
* PROJECT_STRUCTURE.txt:
    - (هذا الملف) الدليل التفصيلي لهيكلية الملفات.
* .gitignore:
    - يحدد الملفات التي لا يتم رفعها لـ GitHub (مثل مكتبات node_modules الثقيلة).

2. مجلد الأصول (public/)
------------------------
* يحتوي على ملفات ثابتة لا تتغير برمجياً.
* الصور الأساسية، الأيقونات (Icons)، وشعار الموقع (Logo).
* ملف favicon.ico الذي يظهر في لسان المتصفح.

3. مجلد الكود المصدري (src/) - "قلب المشروع"
------------------------------------------
أ. الملفات الأساسية:
* main.tsx: يربط كود React بملف الـ HTML الأساسي.
* App.tsx: يمثل "مركز التحكم والتوجيه" (Main Router)، حيث يتم تعريف كافة مسارات الموقع (مثل /hotels, /booking).

ب. مجلد التطبيق والصفحات (src/app/):
* page.tsx: الصفحة الرئيسية الجذابة (Home Page).
* globals.css: يحتوي على تنسيقات CSS العالمية وخطوط الموقع.
* /hotels: صفحات استعراض الفنادق والبحث.
* /hotels/view: صفحة تفاصيل الفندق الفردية (تم بناؤها لتكون تفاعلية بالكامل).
* /booking: صفحات عملية الحجز (اختيار التواريخ، تفاصيل الضيوف).
* /dashboard: لوحة تحكم المستخدم لمتابعة حجوزاته.
* /admin: واجهات محاكاة للمدير لمراقبة النظام.
* /owner: واجهات مخصصة لأصحاب الفنادق لإدارة عقاراتهم.
* /settings & /profile: إعدادات المستخدم الشخصية.

ج. مجلد المكونات (src/components/):
* /ui: مكونات صغيرة وذرية (Buttons, Inputs, Badges, Cards) مبنية باحترافية.
* /layout: الهيكل الثابت للموقع (Navbar العلوي، Footer السفلي، BaseLayout).
* /sections: أقسام الصفحات الكبيرة مثل (Hero Section, Featured Hotels, Testimonials).
* /hotel: مكونات متخصصة لعرض بيانات الفندق (معرض الصور، قائمة الخدمات، خريطة الموقع).
* /dashboard: مكونات مخصصة لواجهة لوحة التحكم (إحصائيات، جداول).

د. مجلد سياق البيانات (src/context/):
* starlux-context.tsx: 
    - المسؤول عن "ذاكرة التطبيق".
    - يحفظ بيانات المستخدم المسجل، الفنادق المفضلة، وعمليات الحجز النشطة بشكل موحد لكل الصفحات.

هـ. مجلد الأدوات والبيانات (src/lib/):
* mock-data.ts: 
    - "قاعدة البيانات الوهمية". تحتوي على بيانات مئات الفنادق، الصور، والأسعار.
* db.ts: 
    - المحرك الذي يحاكي قاعدة البيانات الحقيقية.
    - يقوم بحفظ بياناتك في متصفحك (LocalStorage) لتبقى محفوظة حتى بعد إغلاق المتصفح.
* utils.ts: وظائف مساعدة برمجية لتنسيق التواريخ والعملات.

4. مجلد البناء النهائي (dist/)
-----------------------------
* يحتوي على النسخة "المصغرة والمضغوطة" من المشروع الجاهزة للنشر على الإنترنت.
* تم إدراجه في الرفع ليكون المستودع كاملاً وجاهزاً للمعاينة الفورية.

خلاصة:
------
المشروع مصمم ليكون "Frontend-only" ولكن بذكاء "Full-stack"، حيث يحاكي وجود قاعدة بيانات وسيرفر عبر تقنيات المتصفح الحديثة، مما يجعله مثالياً للاستعراض (Demo) دون الحاجة لإعدادات معقدة.
